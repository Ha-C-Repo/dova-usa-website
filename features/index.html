Every Feature Built for Accountability

DOVA doesn't just make vehicle access easier. It makes every access event provable, documented, and legally defensible.

Core Patent Asset

The Only System That Requires All Three to Pass — In Order.

Most access systems check one thing: did this person authenticate? DOVA checks three, in sequence, before a single command leaves the cloud. Two-factor identity verification. Role-based permission confirmation for that specific vehicle and command. Manager approval where required. The vehicle command is not dispatched until all three gates clear. There is no shortcut. There is no bypass.

Why It Matters: Machine-enforced accountability. Every access event is identity-verified, role-confirmed, and supervisor-authorized — on record, permanently.
Crown Jewel IP

The Access Log That Answers Every Dispute.

At the exact moment an access event is authorized, DOVA's firmware captures a full OBD-II diagnostic snapshot — odometer, fuel level, battery state of charge, tire pressure, and active fault codes. That snapshot is cryptographically bound to the specific authorization transaction and written to a write-once, append-only log. It cannot be modified. It cannot be deleted. It can be exported for legal proceedings, insurance claims, or fleet audit compliance.

Why It Matters: When an employee claims they weren't responsible for the scratch, the missing fuel, or the extra miles — you have a cryptographic record of the vehicle's exact condition at every prior access event.
No OEM Required

One Module. Every Vehicle Since 1996.

The OBD-II port is federally mandated on every U.S. passenger and light commercial vehicle manufactured since 1996. It is physically identical on a Toyota, a Ford, a BMW, and a Chevrolet. DOVA connects through this universal interface — without OEM API agreements, without infotainment system access, without make/model-specific hardware programming. Your fleet is compatible on day one, regardless of vehicle mix.

Why It Matters: No OEM agreements. No proprietary integrations. No per-brand licensing. One platform for your entire fleet.
Security Architecture

No Bluetooth. No NFC. No Proximity Tricks.

All commands from the DOVA cloud to the hardware module travel exclusively via 4G LTE / 5G cellular. The hardware module firmware does not implement a Bluetooth, NFC, or short-range wireless command interface. This is a deliberate, locked architectural decision — not a configuration option. No one can bypass the cloud authorization engine by standing next to the vehicle with a device.

Why It Matters: Every command passes through the cloud. Every command passes through all three gates. No exceptions.

Every Vehicle. Live Position. No Guessing.

The DOVA fleet map shows the real-time GPS location of every enrolled vehicle. Track test drives. Monitor vehicle movement after hours. Confirm vehicle location at the time of any access event.

You Define the Boundary. DOVA Enforces It.

Administrator-defined geographic zones per vehicle or vehicle group. DOVA evaluates both the user's GPS location and the vehicle's GPS position before advancing any authorization. If either is outside the permitted zone, access is denied. Geofence evaluation is performed server-side — never by the mobile app.

Unauthorized Removal Doesn't Go Unnoticed.

If the DOVA module is physically removed or loses power without authorization, the firmware transmits a cryptographically timestamped alert — including last-known GPS coordinates and device state — to the cloud before shutdown. The event is permanently logged.

Four Levels. Fully Configurable.

System Administrator → Fleet Manager → Authorized Operator → Read-Only Observer. Configurable permissions govern access requests, approvals, diagnostic viewing, geofence editing, and log export rights — per role, per vehicle.
