About this prototype

  • This is the Phase 2 prototype. The production site is at dovausa.com. The Phase 1.5 prototype (AI-imagery + scroll crossfade) is at /prototype/ for comparison.
  • Real WebGL 3D scene rendered with Three.js (no static images on the hero).
  • HDR environment lighting via PMREMGenerator + Three.js RoomEnvironment (procedural studio environment - no external HDR file dependency, no CORS issues).
  • UnrealBloomPass postprocessing produces real glow on emissive surfaces (the cyan LED, the cloud authorization sphere).
  • PBR materials with metalness, roughness, and clearcoat on the module and port.
  • GSAP ScrollTrigger pins the hero and drives a cinematic camera path through four keyframes (Install → Enroll → Authorize → Network).
  • Lenis smooth-scroll inertia for the app-like feel.
  • Mobile fallback: the 3D scene is replaced with the Phase 1.5 cinematic image (3D on small screens degrades quality and battery).
  • The next step beyond this is a 3D artist contractor producing bespoke Blender models of the OBD-II port and a real vehicle. Budget $2-5k, 3-5 days.
